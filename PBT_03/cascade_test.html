Bài C2 – Cascade Puzzle

Phân tích thứ tự ưu tiên trong CSS Cascade

Bước 1

CSS Rules

Dưới đây là bộ quy tắc CSS đang được áp dụng cho puzzle.

body { font-size: 16px; color: #333; }
.container { font-size: 14px; }
.card { color: blue; }
.card .title { font-size: 20px; }
.card p { color: inherit; }
#featured .title { color: red; }
.highlight { color: green !important; }

Bước 3

Bảng phân tích Cascade

Giải thích chi tiết lý do mỗi phần tử nhận giá trị computed cuối cùng.

Phần tử Font-size Color Giải thích Cascade
Sản phẩm A h2.title.highlight trong #featured 20px green !important
font-size: .card .title đặt font-size: 20px (specificity 0-2-0), ghi đè giá trị 14px kế thừa từ .container.
color: Có 2 rule cạnh tranh — #featured .title (0-1-1-0 → red) và .highlight (0-0-1-0 → green). Dù #featured .title có specificity cao hơn, .highlight dùng !important nên green thắng.
Mô tả sản phẩm p trong #featured 14px blue Inherit
font-size: Không có rule nào đặt trực tiếp cho p. Kế thừa từ .container → 14px.
color: .card p đặt color: inherit, buộc p kế thừa giá trị color từ phần tử cha .card. Vì .card { color: blue } → blue.
Sản phẩm B h2.title trong .card 20px blue Specificity
font-size: .card .title đặt font-size: 20px (specificity 0-2-0).
color: Không có rule nào đặt color trực tiếp cho .card .title (card này không có id="featured"). Phần tử h2 không tự động kế thừa color vì user-agent stylesheet đặt color riêng, nhưng ở đây không có giá trị nào khác ngoài giá trị kế thừa từ .card { color: blue } → blue.
Mô tả sản phẩm B p.highlight trong .card 14px green !important
font-size: Kế thừa từ .container → 14px.
color: Có 2 rule — .card p (0-1-1 → inherit → blue) và .highlight (0-1-0 → green !important). Dù .card p có specificity tương đương, .highlight dùng !important nên green thắng.
